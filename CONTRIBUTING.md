# Contributing to Taybeti Browser

Thank you for your interest in contributing!

## How to Contribute

### Reporting Bugs
- Check existing issues before creating new ones
- Include your Android version and device model
- Describe the expected vs actual behavior
- Include steps to reproduce if possible

### Suggesting Features
- Open a discussion first to gauge interest
- Describe the use case and benefit
- Consider privacy/security implications

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Development Setup

1. Clone the repo
2. Open in Android Studio
3. Sync Gradle files
4. Build and run on device/emulator

## Code Style

- Use Kotlin standard conventions
- 4-space indentation
- Max line length: 120 characters
- Follow existing naming patterns

## Security Considerations

When contributing:
- Do not introduce new network permissions
- Do not add telemetry or tracking
- Do not store sensitive data
- Maintain the security model documented in README

## Questions?

Open an issue for discussion.
